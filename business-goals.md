# Autonomous Arcade — Business Goals

**North Star:** $50,000 MRR via AdSense

**Primary metric:** Monthly AdSense revenue
**Secondary:** Monthly unique visitors, pageviews, session duration

---

## Revenue Model

- AdSense is the first revenue stream
- Revenue = impressions × CTR × CPM
- Higher traffic + longer sessions = more revenue

---

## Growth Levers (Priority Order)

### 1. Traffic (get users to the site)
- SEO: sitemap, structured data, page speed, meta tags
- Shareability: each game must be easy to share (URL, screenshot, score card)
- New games published frequently (current: every 2 hours)
- Social proof: activity log shows the site is alive and improving

### 2. Retention (keep users on the site)
- Games must load fast, run smooth, and be immediately engaging
- Clear home page / navigation so users don't get stuck
- Feedback loop: users see their feedback results in real changes (changelog)

### 3. Monetization (turn traffic into revenue)
- AdSense integration on all pages
- Ad placement that doesn't interrupt gameplay
- More pageviews per session = more ad impressions

---

## What This Means for Game Changes

When evaluating feedback, ask:
- Does this change bring back more users? (retention)
- Does this change get more users to share? (traffic)
- Does this change keep users on the page longer? (monetization)
- Does this fix something that causes users to leave? (retention)

**High impact changes:** Speed, load time, shareability, first 5-second experience, broken/annoying bugs
**Low impact changes:** Cosmetic tweaks that don't affect engagement

---

## Non-Goals

- Don't add features that slow page load
- Don't make games frustratingly hard (kills retention)
- Don't interrupt gameplay with ads
- Don't chase individual user requests blindly — evaluate through the lens above
